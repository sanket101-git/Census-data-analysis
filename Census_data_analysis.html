1.2 Data Discripction

The dataset is a Census data. It contains 32561 rows and 15 columns.

The target variable is y representing income levels. The target variable is categorical as it just shows as less than or greater than 50k. The goal is to predict whether an individual’s income is above or below $50,000 based on the other features.

Following are the models that were covered in class C5.0
naiveBayes rpart lm M5P MLP
ksvm IBk SimpleKMeans apriori

C5.0: C5.0 (and its predecessor, C4.5) is a decision tree algorithm that can handle both categorical and numerical data. It’s suitable for classification tasks for predicting income levels (<=50K or >50K).

naiveBayes: Naive Bayes is a probabilistic algorithm that assumes independence between predictors.It can also be used

rpart: Recursive Partitioning (rpart) is another decision tree algorithm. It’s suitable for classification and can handle both numeric and categorical variables.It can be used

ksvm: Support Vector Machines (SVM) can handle both classification and regression tasks. It’s effective in high-dimensional spaces and can handle non-linear relationships. It might work well if there are complex decision boundaries.

IBK:-

The models that cannot be used are:-

SimpleKMeans: K-Means is an unsupervised clustering algorithm, and it’s not suitable for a supervised classification task like predicting income levels.

apriori: Apriori is an algorithm used for association rule mining in transaction databases. It’s not appropriate for a classification task.

  {
  set.seed(seedVal)

  cv_results <- lapply(folds, function(x)
  { 
    train <- df[-x,-target]

    train_target <- df[-x,target]
    test_target <- df[x,target]
    
    classification_model <- classification(train,train_target)

cv_function1 <- function(df, target, nFolds, seedVal, prediction_method, metrics_list)
{
  # create folds
  set.seed(seedVal)
  folds = createFolds(df[,target],nFolds) 
  # perform cross validation
  cv_results <- lapply(folds, function(x)
  { 
    test_target <- df[x,target]
    test_input  <- df[x,-target]

    train_target <- df[-x,target]
    train_input <- df[-x,-target]

    prediction_model <- prediction_method(train_target~.,train_input) 
    pred<- predict(prediction_model,test_input)
    return(mmetric(test_target,pred,metrics_list))
  })
  # generate means and sds and show cv results, means and sds using kable
  #cv_results_m <- as.matrix(as.data.frame(lapply(cv_results, as.character)))
  cv_results_m <- as.matrix(as.data.frame(cv_results))
  cv_mean<- as.matrix(rowMeans(cv_results_m))
 # print(class(cv_results_m))
  #str(cv_results_m)
  #print(cv_mean)
  #cv_results_m_numeric <- as.numeric(cv_results_m)
  cv_sd <- as.matrix(rowSds(cv_results_m))
  colnames(cv_mean) <- "Mean"
  colnames(cv_sd) <- "Sd"
  cv_all <- cbind(cv_results_m, cv_mean, cv_sd)
  kable(t(cv_all),digits=2)
}

This can result in the person experiencing financial hardship, possible loan defaults, and unfavorable outcomes including lowered credit ratings. Effect on Enterprise: This may lead to higher rates of default, more financial risk, and possible losses for the financial institution. Additionally, if clients feel mislead or go into financial troubles as a result of unsuitable product offerings, it could damage the institution’s reputation. 2. Selling Less Profitable Goods to People with High Incomes:Consequences: If a high-earner is sold a less lucrative financial product because it was miscalculated that they would have a low income, there could be lost revenue potential. The person may be qualified for and eager to invest in more profitable financial goods, but the mistake in the marketing keeps them from having access to these choices. Impact on Business: If high-value clients are not targeted with suitable products, the financial institution may see a decline in revenue and profitability. It could also lose out on potential long-term clients who could have made a big difference in the institution’s performance. Predictive model accuracy is critical in both scenarios to ensure that marketing efforts match customers true financial capabilities and preferences.
